Proposed redesign

A warmer, sharper judging system built around decisions instead of generic cards.

The redesign keeps the same routes and product intent, but changes the visual language to a single accent, higher-contrast hierarchy, and structures that match each task: triage on the directory, evidence on project detail, calibration on scoring, escalation on results, and readiness on asset upload.

Applied from taste-skill

Single accent palette, stronger type character, fewer generic cards, asymmetry where it helps, and dashboard-safe sans-serif typography.

Applied from redesign-skill

Priority order: font swap, palette cleanup, state clarity, layout refinement, then component replacement for the most generic surfaces.

Applied from output-skill

All five major prototype screens are represented as complete standalone mockups rather than partial fragments.

Screen 1

Directory becomes a triage desk

Instead of treating every project as the same card, the redesigned directory splits the screen into a status rail and a priority queue. That makes urgent work and incomplete judging more legible than cover imagery.

Single accent and warmer neutrals The teal and purple mix is replaced with one copper accent so statuses and calls to action feel intentional.
Status-first layout Filters move into a command rail and projects are rendered as ranked rows with urgency tags instead of equal-weight cards.
More useful metrics Completion, average score, and pending tie-breaks sit in the same visual plane as the queue instead of being pushed to the footer.

Screen 2

Project detail becomes an evidence brief

The project page should not just impress; it should help a judge decide. The redesign shifts the hero from pure mood into a summary brief, then keeps track fit, readiness, and team evidence visible beside the story.

Hero supports judgment The headline shares space with a summary of readiness, strongest differentiator, and route to scoring.
Metadata becomes a briefing rail Booth, track, team, and reviewer notes are grouped as operational context rather than a secondary card.
Evidence before action The action layer is separated from supporting proof so the page encourages informed scoring rather than impulsive clicking.

Screen 3

Scoring workbench becomes a calibration surface

Rather than a stack of interchangeable cards, the scoring page exposes the rubric as a comparative system. Judges can see what is strong, what is incomplete, and how confident the current draft feels before submitting.

Comparative rubric blocks Each criterion shares a consistent row structure so judges can scan relative strength instead of re-learning each card.
Decision summary rail The right side now shows draft total, missing sections, and review confidence before the commit action.
Explicit note framing Qualitative feedback becomes guided evidence capture, not a placeholder-only input field.

Screen 4

Results dashboard becomes an escalation console

The current results table is readable, but the redesigned version separates ranking from intervention. Tie cases are promoted to top-level blocks while the finalist ladder stays crisp and scan-friendly beneath.

Tie-breaks are first-class Urgent decisions sit above the leaderboard with explicit case framing and next actions.
Leaderboard loses the generic table shell A ladder format reads faster in review sessions and creates clearer emphasis for finalists and borderline qualifiers.
Status colors stay within one palette The status system uses accent, muted neutral, and warning warmth instead of introducing purple as a second design language.

Screen 5

Asset upload becomes a readiness workshop

The upload flow already has the right split-screen idea. The redesign strengthens it by turning the left side into a checklist, the center into structured asset capture, and the right side into a judge-facing outcome preview.

Checklist before media Participants see what matters to judges before they start uploading files, which improves intent and completion.
Form framing replaces generic dropzones Every asset block explains what the judge learns from that asset, not only its file format.
Preview rail becomes interpretive The right rail shows how the page will read to judges and what narrative gaps still remain.
